Úvod

Tento dokument slouží jako popis semestrálního projektu na předmět ETE87E Značkovací jazyky kombinované studium v ročníku 2016/2017. Součástí dokumentu je popis vývoje semestrální práce obsahující tvorbu XML dokumentu, XSD schéma, XSL transformace do HTML a tato dokumentace. Tématem mé práce je má záliba ve sportu a zvolil jsem vytvoření seznamu fotbalových týmů.

Metodika

Pro tvorbu souborů jsem používal dva programy, jeden XML Copy Editor pro validaci a transformaci souborů, druhým byl EditPlus, který mi sloužil k samotnému psaní souborů a k náhledu XML v zapouzdřeném prohlížeči (protože náhled v XML Copy Editoru mi nefungoval).

Jako první jsem si zkusil vytvořit XSD soubor, kde jsem definoval jednotlivé elementy a atributy. Následně jsem začal psát XML a podle vymyšlené struktury jsem se pokoušel naplnit XML daty. Při tomto postupu jsem narážel na zapomenutá lomítka apod. V některých situacích jsem si již nebyl schopen poradit a provedl jsem v XML Copy Editoru vygenerování XSD z XML a poté jsem porovnával výsledek s mým původním návrhem. Výhodou vygenerování XSD je, že je plně korektní, ale nevýhodou, že neobsahuje mé kontroly a výčty. Proto jsem zachoval svůj původní XSD a pouze jsem identifikoval chybné zápisy. Pro validaci jsem použil jak zabudovanou validaci v XML Copy Editoru, tak zároveň online nástroj na adrese http://www.utilities-online.info/xsdvalidation

Po validaci XSD s XML dohromady jsem začal vytvářet XSL transformaci. Zároveň jsem musel vytvořit i CSS soubor pro formátování tabulek primárně, jelikož zobrazení ve většině používá různé tabulky. Vývoj XSL jsem prováděl tak, že jsem vždy vytvořil koncept určité části dat a otestoval rovnou na XML. Pkud bylo něco chybně, opravil jsem to a pokračoval. Tímto způsobem jsem vyhotovil kostru. Následně jsem si vytvořil CSS soubor a testoval jsem výsledek, přičemž jsem vždy postupně upravoval jednotlivé části. V průběhu vývoje XSL jsem musel párkrát upravit samotnou strukturu XML i XSD, abych např. byly použitelené klubové barvy. Výsledek jsem transformoval do HTML souboru.

Popis elementů a atributů

Položka Typ Popis
fotbalove_tymy Element Element, který obsahuje seznam všech týmů první ligy.
autor Atribut Moje jméno jako autora souboru.
tym Element Konkrétní tým v lize.
ID Atribut Unikátní identifikátor týmu.
jmeno_tymu Element Obsahuje jméno týmu.
zeme Element Obsahuje třímístný kód země, odkud tým pochází. Tento kód je validován na délku 3 znaky.
adresa Element Element s odkazem na komplexní typ "adresa", který je použitý i v dalších částech.
ulice Element Element je součástí elementu adresa a obsahuje jméno ulice. Zároveň obsahuje dva atributy čísla popisného a čísla orientačního.
cislo_popisne Atribut Obsahuje číslo popisné.
cislo_orientacni Atribut Nepovinný údaj o čísle orientačním.
mesto Element Součástí adresy a obsahuje město.
psc Element Součástí adresy a obsahuje PSČ dané adresy. Má vytvořenou kontrolu, že nemůže být prázdný a zároveň je vytvořen formát pomocí regulárního výrazu.
web_adresa Element Element obsahuje odkaz na webové stránky týmu.
tymove_barvy Element Komplexní element použitý pro seznam týmových barev.
barva Element Element jednotlivých barev, přičemž počet barev je neomenezený. Obsahuje jeden atribut.
poradi Atribut Atribut určující jakési ID barvy v rámci všech barev týmu.
jmeno_barvy Element Tento element obsahuje několik atributů a hlavně jméno barvy, které je použito v seznamu barev ve výsledném HTML.
red Atribut Kód R z RGB škály v desítkovém tvaru pro konkrétní barvu.
green Atribut Kód G z RGB škály v desítkovém tvaru pro konkrétní barvu.
blue Atribut Kód B z RGB škály v desítkovém tvaru pro konkrétní barvu.
hraci_tymu Element Komplexní element obsahující seznam všech hráčů v týmu.
hrac Element Element konkrétního hráče, kterých může být v týmu neomezeně. Obsahuje jeden atribut.
cislo_dresu Atribut Atribut hráče, který obsahuje číslo dresu.
jmeno Element Element obsahuje jméno hráče.
prijmeni Element Element obsahuje přjmení hráče.
narodnost Element Element je řetězcová reprezentace národnosti.
narozeni Element Element typu datum, který obsahuje datum narození hráče.
pozice Element Element omezený na seznam konkrétních hodnot pomocí jednoduchého typu s enumerátory.
adresa Element Element obsahující odkaz na komplexní typ adresy (více informací zde adresa).
pocatek Element Element typu datum obsahující datum nástupu hráče do týmu.
historie Element Komplexní element (pomocí reference) obsahující hráčskou historii, kde je v tuto chvíli implementována historie angažmá hráče v předchozích týmech, které je reprezentováno samostatným elementem se čtyřmi atributy.
predchozi_angazma Element Komplexní typ elementu obsahující název předchozího týmu a čtyři atributy s detaily.
pocatek Atribut Datový typ atributu obsahující datum počátku daného angažmá.
konec Atribut Datový typ atributu obsahující datum konce daného angažmá.
golu Atribut Číselný typ atributu obsahující počet gólů, které hráč vstřelil během daného angažmá.
pozice Atribut Atribut požice odkazující na typ enumerátoru hráčských pozic.

Popis transformace

Samotnou transformaci do HTML jsem použil XML Copy Editor. Díky provázanosti XSD, XML a XSL byl schopný editor jedním klikem provést transformaci do finálního HTML souboru. Samozřejmě bylo nutné, aby všechny vazby prošly validací na Well-formedness a následně bylo možné transformaci provést.

Závěr

Díky této semestrální práci jsem se naučil chápat podstatu XSD a XSL (XML a HTML byly soubory, se kterými jsem již mnohokrát pracoval při mém zaměstnání). V jednom z mých předchozích zaměstnání jsem vytvořil parser, který formátoval XML do použitelného tvaru pomocí definice rozdělení jednotlivých elementů a atributů do objektů na formuláři typu textbox, combobox, listtable apod. Nyní je mi jasné, že transformace pomocí XSL je mnohem účinější, flexibilnější a rychlejší řešení.
